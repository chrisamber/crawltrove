# CrawlTrove v0.4 Durable Crawl Core Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace the process-local crawl frontier with a PostgreSQL-backed, restart-safe crawl service that is useful on one host before any remote or managed acquisition work lands.

**Architecture:** FastAPI submits immutable crawl configuration to PostgreSQL and returns immediately. A separate worker loop claims fenced tasks, runs the existing `WebScraper` extraction path, persists one terminal result, and discovers links transactionally; a maintenance loop reclaims leases and finalizes jobs. The legacy file checkpoint reader remains available only through `/api/crawl/{job_id}/resume` for the v0.4 compatibility cycle.

**Tech Stack:** Python 3.11, FastAPI, Pydantic 2, asyncpg, PostgreSQL 16, curl_cffi, Playwright, pytest/pytest-asyncio.

## Global Constraints

- Keep `POST /api/scrape` operational when `DATABASE_URL` is unset; `POST /api/crawl` and durable crawl reads MUST return HTTP 503 instead of silently using memory.
- Retain public crawl limits of 100 unique pages and depth 5.
- Default to four total attempts per URL, at most two attempts per acquisition route, with exponential full-jitter backoff and `Retry-After` honored.
- Default task lease is 120 seconds and heartbeat interval is 30 seconds; every completion MUST match the current lease token.
- Default job timeout is 21,600 seconds; queued, leased, retrying, and waiting tasks all count toward it.
- Default whole-job ceilings are 1 GiB downloaded data and 2 GiB referenced artifacts; callers may lower them but cannot exceed operator caps.
- Enforce decoded response limits of 10 MiB HTML, 50 MiB PDF/EPUB, 25 MiB images, 10 MiB serialized DOM, and 20 MiB screenshots.
- Enforce global cross-job origin concurrency 1 and `next_request_at` in PostgreSQL; no worker may sleep while holding a lease solely for politeness delay.
- Use additive forward-only SQL migrations; do not add Alembic or destructive migrations.
- Keep durable queue operations in `app/crawl/repository.py` and fail closed. Do not add them to best-effort `app/db/repo.py`, whose existing callers intentionally swallow persistence errors.
- Preserve the current single-operator authentication model; API keys do not create tenants.
- Never point `TEST_PG_ADMIN_DSN`, `TEST_DATABASE_URL`, or `TEST_DB_NAME` at production.
- Do not commit generated `data/`, `tmp/`, `.pytest_cache/`, or benchmark output.

- [ ] **Step 3: Implement one-transaction submission**

Use `uuid.uuid4()`, SHA-256 of the normalized URL, and `urlsplit` to build `origin_key`. In one transaction, insert the job and seed task; on `idempotency_key` conflict, return the existing job without adding another task:

```python
async def submit_job(config: CrawlConfig, idempotency_key: str | None = None,
                     definition_job_id: int | None = None,
                     trigger: str = "manual") -> UUID:
    pool = await require_pool()
    normalized = normalize.normalize_url(config.url)
    job_id, task_id = uuid.uuid4(), uuid.uuid4()
    parsed = urlsplit(normalized)
    port = parsed.port or (443 if parsed.scheme == "https" else 80)
    origin_key = f"{parsed.scheme}://{parsed.hostname}:{port}"
    url_hash = hashlib.sha256(normalized.encode("utf-8")).digest()
    async with pool.acquire() as conn, conn.transaction():
        if idempotency_key:
            existing = await conn.fetchval(
                "SELECT id FROM crawl_jobs WHERE idempotency_key = $1",
                idempotency_key,
            )
            if existing:
                return existing
        run_id = await conn.fetchval(
            """INSERT INTO scrape_runs(external_id,job_id,trigger,status)
               VALUES ($1,$2,$3,'pending') RETURNING id""",
            str(job_id), definition_job_id, trigger,
        )
        await conn.execute(
            """INSERT INTO crawl_jobs
               (id,run_id,state,config,max_pages,max_bytes,max_artifact_bytes,
                discovered_count,deadline_at,idempotency_key)
               VALUES ($1,$2,'pending',$3::jsonb,$4,$5,$6,1,
                       now() + make_interval(secs => $7),$8)""",
            job_id, run_id, config.model_dump_json(), config.limit, config.maxBytes,
            config.maxArtifactBytes, config.timeoutSeconds, idempotency_key,
        )
        await conn.execute(
            """INSERT INTO crawl_tasks
               (id,job_id,original_url,normalized_url,url_hash,origin_key,depth,
                discovery_seq,state,max_attempts)
               VALUES ($1,$2,$3,$4,$5,$6,0,0,'pending',$7)""",
            task_id, job_id, config.url, normalized, url_hash, origin_key,
            config.acquisition.maxAttempts,
        )
        await conn.execute(
            "INSERT INTO crawl_origins(origin_key) VALUES ($1) ON CONFLICT DO NOTHING",
            origin_key,
        )
        await _event(conn, job_id, task_id, "job_submitted", {})
    return job_id
```

For a concurrent idempotency-key unique conflict, roll back the attempted inserts and select/return the winning job ID in a fresh transaction. `require_pool()` raises `PersistenceUnavailable` when `get_pool()` returns `None`; only the route translates it to 503.

```python
# app/crawl/classify.py
from dataclasses import dataclass
import random


@dataclass(frozen=True)
class FailureDecision:
    retry: bool
    error_class: str
    error_code: str


def classify_failure(reason: str | None, status: int | None) -> FailureDecision:
    if reason in {"unsafe_url", "blocked_robots", "policy_error", "invalid_input"}:
        return FailureDecision(False, "policy", reason or "policy_error")
    if status == 429 or (status is not None and 500 <= status <= 599):
        return FailureDecision(True, "http", f"http_{status}")
    if reason in {"transport_error", "timeout", "dns_error", "tls_error", "lease_expired"}:
        return FailureDecision(True, "transport", reason)
    if status is not None and 400 <= status <= 499:
        return FailureDecision(False, "http", f"http_{status}")
    return FailureDecision(False, "permanent", reason or "unknown_failure")


def backoff_seconds(attempt: int, retry_after: float | None = None) -> float:
    if retry_after is not None:
        return max(0.0, retry_after)
    ceiling = min(60.0, 2 ** max(0, attempt - 1))
    return random.uniform(0.0, ceiling)
```

## Plan 1 Exit Gate

- A crawl accepted by one API process is readable after that process restarts.
- Killing the worker after fetch but before completion leads to a later fenced retry and one persisted result.
- `/api/crawl` never runs process-local work and returns 503 without PostgreSQL.
- `/api/scrape` retains its current no-database behavior and response fields.
- The legacy checkpoint resume endpoint remains available for v0.4 only.
- No task can exceed page, depth, deadline, byte, artifact, browser, retry, or origin-politeness limits.
