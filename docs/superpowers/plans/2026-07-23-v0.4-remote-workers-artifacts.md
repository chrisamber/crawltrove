# CrawlTrove v0.4 Remote Workers and Artifact Storage Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Run durable acquisition workers on other hosts without granting them general database or object-store access.

**Architecture:** Workers connect directly to PostgreSQL with an individually enrolled login that can execute only a narrow `worker_api` schema. Core and workers exchange immutable content-addressed references through S3-compatible storage; every worker credential is restricted to `workers/<worker-id>/`. The same worker images run in local Compose and remotely, while local Compose may explicitly allow plaintext PostgreSQL on its private network.

**Tech Stack:** Python 3.11, asyncpg, PostgreSQL 16 roles and `SECURITY DEFINER` functions, boto3/S3-compatible storage, TLS client certificates, Docker multi-stage builds, Docker Compose, MinIO.

## Global Constraints

- Plan 1 is merged and its migration, `CrawlConfig`, `ClaimedTask`, `TaskResult`, and fenced transition semantics are stable inputs.
- A remote worker has one PostgreSQL login, one `workers` row, one capability set, and one object prefix; identities are individually revocable and auditable.
- The worker role has no direct table privileges and may execute only claim, heartbeat, complete, fail, wait, and health functions in `worker_api`.
- Remote PostgreSQL requires `sslmode=verify-full` semantics and an enrolled client certificate; insecure DB transport requires explicit `WORKER_ALLOW_INSECURE_DB=true` and is reported degraded.
- Artifact keys are immutable SHA-256 addresses and completion commits URI, size, checksum, and media type under the current lease token.
- Filesystem artifacts remain valid only when API and worker are on one host with a shared volume; multi-host mode fails startup unless S3 is configured.
- Every worker S3 credential is scoped to `workers/<worker-id>/`; workers cannot read or write another worker's prefix.
- Server-side encryption is enabled and abandoned temporary uploads are deleted after 24 hours; referenced objects are not deleted automatically by default.
- Worker protocol mismatches prevent new claims; an older compatible worker may finish its current lease and drain.
- Use the same 120-second task lease and 30-second heartbeat fixed in Plan 1.
- Do not add Kubernetes charts, Helm, Terraform, or a second package manager.

```python
# app/artifacts/base.py
from dataclasses import dataclass
from typing import AsyncIterable, Protocol


class ArtifactTooLarge(ValueError):
    pass


class ArtifactIntegrityError(ValueError):
    pass


@dataclass(frozen=True)
class ArtifactRef:
    uri: str
    size: int
    sha256: str
    media_type: str


class ArtifactStore(Protocol):
    async def put(self, chunks: AsyncIterable[bytes], media_type: str,
                  expected_max_bytes: int) -> ArtifactRef:
        raise NotImplementedError
    async def get(self, ref: ArtifactRef) -> bytes:
        raise NotImplementedError
    async def delete(self, ref: ArtifactRef) -> None:
        raise NotImplementedError
    async def exists(self, ref: ArtifactRef) -> bool:
        raise NotImplementedError
    async def verify(self, ref: ArtifactRef) -> bool:
        raise NotImplementedError
```

- [ ] **Step 4: Implement atomic filesystem publication**

`FilesystemArtifactStore.put()` writes chunks to a UUID-named file under `<root>/tmp`, updates SHA-256 and byte count before each write, raises before writing the chunk that exceeds `expected_max_bytes`, fsyncs, then publishes with `os.replace` to `sha256/<first-two>/<digest>`. If the digest already exists, delete only the temporary file. `get()` and `verify()` reject URIs outside the configured root using `Path.resolve().is_relative_to(root.resolve())`.

- [ ] **Step 3: Implement S3 publication and verification**

Stage each bounded stream to `tempfile.SpooledTemporaryFile(max_size=8 * 1024 * 1024)`, hash while streaming, then call boto3 through `asyncio.to_thread`. Upload first to `workers/<id>/tmp/<uuid>`, copy it to the immutable SHA key with `ServerSideEncryption` set from `S3_SERVER_SIDE_ENCRYPTION` (default `AES256`), then delete the temporary object. The constructor accepts only worker IDs matching `^[a-zA-Z0-9][a-zA-Z0-9_-]{0,63}$`; every operation derives the key itself and rejects foreign `s3://` references.

- [ ] **Step 3: Add worker identities and a locked function schema**

```sql
CREATE TABLE workers (
    id TEXT PRIMARY KEY,
    db_role NAME UNIQUE NOT NULL,
    capabilities TEXT[] NOT NULL DEFAULT '{}',
    protocol_version INTEGER NOT NULL,
    state TEXT NOT NULL CHECK (state IN ('active','draining','revoked','incompatible')),
    artifact_prefix TEXT UNIQUE NOT NULL,
    last_seen_at TIMESTAMPTZ,
    created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
    revoked_at TIMESTAMPTZ
);

REVOKE ALL ON workers FROM PUBLIC;
CREATE SCHEMA worker_api;
REVOKE ALL ON SCHEMA worker_api FROM PUBLIC;
```

Every function is `SECURITY DEFINER SET search_path = pg_catalog, public`, checks `workers.db_role = session_user`, `state = 'active'`, and exact protocol compatibility before touching a task. Revoke public execution in the migration. The admin-only enrollment command creates the `crawltrove_worker` NOLOGIN role when absent, grants it schema usage/function execution, and creates individual LOGIN INHERIT roles. This keeps ordinary application migrations compatible with external databases where the application owner lacks `CREATEROLE`.

## Plan 2 Exit Gate

- A remote worker can claim, heartbeat, upload, and complete through only restricted DB functions.
- Direct reads or writes to crawl tables fail for every worker login.
- Cross-worker artifact access and foreign-prefix completion fail.
- Remote configuration refuses unverifiable PostgreSQL TLS and filesystem artifacts.
- Revoked or protocol-incompatible workers cannot claim; stale workers cannot commit.
- Default Compose exercises PostgreSQL, S3, standard worker, and browser worker without adding another orchestrator.
