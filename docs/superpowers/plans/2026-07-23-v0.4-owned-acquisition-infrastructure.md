# CrawlTrove v0.4 Owned Acquisition Infrastructure Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add operator-configured and operator-owned proxy egress, authorized image/text CAPTCHA solvers, encrypted reusable profiles, and secure live human intervention without exposing raw browser-control protocols.

**Architecture:** Browser workers keep fresh contexts per task and may persist only encrypted profile state. Lightweight egress agents accept authenticated HTTP CONNECT over mTLS and independently enforce public-network targets. When automation needs a human, the durable task enters `waiting_input`, releases its lease, and points to a separately expiring live session reached through an outbound worker WebSocket tunnel with a narrow action protocol.

**Tech Stack:** Python 3.11 asyncio, FastAPI WebSockets, Playwright, Tesseract/pytesseract, cryptography AES-GCM, PostgreSQL 16, TLS client certificates, Docker Compose profiles.

## Global Constraints

- Plans 1 and 2 are merged; use their fenced tasks, worker identities, restricted DB functions, S3 artifact references, enrollment conventions, and protocol checks.
- Owned proxy nodes are operator-controlled VPS or servers only; do not implement peer, crowdsourced, or residential sourcing.
- Operator-supplied proxy pools and image/text solver endpoints are loaded only from mode-0600 mounted secret files or env references; jobs cannot supply endpoints or credentials.
- Owned egress supports HTTP CONNECT only in v0.4; do not add SOCKS5.
- Core validates the target before dispatch; every egress agent independently resolves the host, rejects private/reserved/loopback/link-local/multicast addresses, connects to one validated address, and reports it.
- Egress authentication uses mTLS and short-lived node credentials; there is no unauthenticated listener.
- First-party CAPTCHA automation handles image/text challenges only on operator-authorized domains.
- reCAPTCHA, hCaptcha, Turnstile, and unauthorized challenges require human intervention or an approved managed provider.
- CAPTCHA authorization is operator-controlled through CLI/env, never job input. Exact hosts match only; `*.example.com` must be explicit, public-suffix wildcards are invalid, and redirects are rechecked.
- Reusable profiles are explicitly created/deleted, domain-allowlisted, encrypted, versioned for key rotation, and pinned to one provider or owned pool.
- Live human intervention defaults to 15 minutes, is operator-configurable from 5–60 minutes, and never outlives the job deadline.
- Human timeout is terminal `human_input_timeout` without automatic retry; keep the reusable encrypted profile but close the live browser.
- Owned browser control uses an authenticated outbound WebSocket and narrow screenshot/click/fill/press/scroll/resume/cancel messages; never expose CDP or VNC.
- Session tokens are single-use, narrowly scoped, expire, and are neither persisted in plaintext nor logged.

- [ ] **Step 3: Implement a bounded HTTP CONNECT parser**

Read no more than 16 KiB through `readuntil(b"\r\n\r\n")`, require method `CONNECT`, HTTP/1.1, one ASCII/IDNA hostname, and configured port 80 or 443. Reject URL credentials, literals with zone IDs, extra request bodies, proxy-forwarding headers, and malformed IPv6 authorities. Resolve with `loop.getaddrinfo(host, port, type=SOCK_STREAM)`; reject the entire request if any candidate is not globally routable.

- [ ] **Step 4: Implement mTLS serving and pinned connection**

Choose one validated resolved address and pass it directly to `asyncio.open_connection`; never resolve the hostname again. Return `HTTP/1.1 200 Connection Established` with `X-CrawlTrove-Connected-IP: <ip>`, then pump at most two bounded 64 KiB buffers until EOF, idle timeout, or configured byte ceiling. Log node ID, destination host hash, connected IP, bytes, and duration; never log credentials or query strings.

The enrollment CLI follows Plan 2 certificate handling, writes an agent bundle mode 0600, and revocation updates the core proxy-node state before revoking its certificate serial in the operator-maintained CRL.

## Plan 3 Exit Gate

- Operator-owned CONNECT nodes independently block unsafe targets and require mTLS.
- Proxy selection is auditable, fenced, health-aware, and contains no peer/residential sourcing.
- First-party automation touches only authorized image/text challenges; token challenges require human or managed provider.
- Profile state is encrypted, domain-pinned, backend-pinned, explicitly managed, and rotatable.
- Waiting for a human consumes neither a task lease nor normal worker claim slot.
- The owned live-session surface exposes no raw CDP, VNC, JavaScript evaluation, or reusable control URL.
- Human timeout/cancel closes the browser, prevents stale completion, and produces a stable terminal result.
